#### Реальные изменения (4 файла — в проекте НОВЕЕ, с кастомными патчами)

**Файл 1: `catalog/controller/extension/shipping/measoft.php`**

В проекте добавлен метод `getWeight()` с конвертацией веса:
```php
private function getWeight()
{
    $weight = $this->cart->getWeight();
    if (!$weight) {
        $weight = 0.1;
    }
    if($this->config->get('config_weight_class_id')==2){
        $weight=$weight/1000;
    }
    return round($weight, 2);
}
```
В пакете: `getWeight()` отсутствует, вес берётся напрямую `$this->cart->getWeight()`.

**Файл 2: `catalog/model/extension/shipping/measoftcourier.php`**

В проекте:
- Добавлен `getWeight()` (аналогично).
- Добавлено определение `paytype` через сессию и `fetchPaytype()`.
- Передача `'paytype' => $paytype` в калькулятор.

В пакете:
- `$weight = $this->cart->getWeight()` напрямую.
- `mass => $weight < 1 ? 1 : $weight` (минимум 1 кг).
- `paytype` НЕ передаётся в калькулятор.

**Файл 3: `catalog/model/extension/shipping/measoftcouriershipping.php`**

В проекте:
- `getWeight()` с конвертацией.
- `paytype` через `fetchPaytype()`.

В пакете:
- `getWeight()` без конвертации.
- `paytype` НЕ передаётся.

**Файл 4: `system/library/measoft/measoftcourier.class.php`**

В проекте:

- В `calculatorRequest()`: `$xml .= $this->makeXMLNode('paytype', ...)`.

В пакете:
- Метод `fetchPaytype()` отсутствует.
- В `calculatorRequest()`: узел `paytype` отсутствует.

**Действие:** НЕ перезаписывать эти 4 файла. Они содержат кастомные патчи, необходимые для проекта. Если в пакете есть другие полезные изменения (например, минимальная масса 1 кг для курьера), их нужно применить вручную точечным edit.

## 4. Риски

### 4.1 Git-деплой modifications
`system/storage/modification/` **включён в git** (не в `.gitignore`). Это значит:
- После применения OCMOD в админке (Extensions → Modifications → Обновить) сгенерированные файлы попадут в `system/storage/modification/`.
- Их нужно **закоммитить**, иначе следующий deploy откатит изменения.
- Этот урок уже зафиксирован в анализе задачи 1.2 (Simple install).

### 4.2 Доступность home.courierexe.ru
Карта ПВЗ зависит от внешнего JS `https://home.courierexe.ru/js/measoft_map.js`. Если домен недоступен с сервера магазина — карта не загрузится. Проверяется вручную через curl или браузер.

### 4.3 Конфликт с SimpleCheckout
OCMOD-патчи для `simplecheckout.php` и `simplecheckout_shipping.twig` зависят от того, установлен ли Simple в проекте. Установлен (задачи 1.2/1.3). Патчи используют `error="skip"`, поэтому если точки внедрения не найдены — патчи просто пропускаются без ошибки.

### 4.4 Несоответствие версий
- `install.xml`: v4.3.12
- PHP-код: v4.3.1
Это норма для данного модуля — автор обновляет OCMOD отдельно от PHP-кода.

## 6. Hotfix 2026-07-22: `showModalMea is not defined` в SimpleCheckout

### 6.1 Симптом
На странице `index.php?route=checkout/simplecheckout` кнопка «Выбрать ПВЗ на карте» выводилась, но клик завершался ошибкой браузера:

```text
Uncaught ReferenceError: showModalMea is not defined
```

### 6.2 Причина
Функция `showModalMea` объявлена в `catalog/view/javascript/measoftcourier/checkout_script.js`, но на странице SimpleCheckout этот файл не подключался. В `measoft_oc3.ocmod/install.xml` есть операция для `catalog/controller/checkout/simplecheckout.php`, которая должна добавлять:

```php
$this->document->addScript('https://home.courierexe.ru/js/measoft_map.js?v4.3.11');
$this->document->addScript('/catalog/view/javascript/measoftcourier/checkout_script.js?v4.3.11');
$this->document->addStyle('/catalog/view/javascript/measoftcourier/style.css', $rel = 'stylesheet');
```

Но в текущем исходном `catalog/controller/checkout/simplecheckout.php` этих строк не было, а сгенерированный `system/storage/modification/catalog/controller/checkout/simplecheckout.php` отсутствовал. Это означает, что OCMOD-вставка для SimpleCheckout не была применена/сгенерирована, хотя HTML кнопки уже приходил из quote-текста модуля.

### 6.3 Решение
Добавить подключение Measoft map/widget scripts напрямую в `catalog/controller/checkout/simplecheckout.php` при включённом `shipping_measoftcourier_status`. Это минимальный fallback, соответствующий исходному OCMOD-патчу модуля.

### 6.4 Риск
Если позже OCMOD будет успешно применён поверх этого же исходника, `Document::addScript()` и `Document::addStyle()` хранят ресурсы по URL-ключу внутри одной позиции, поэтому одинаковые подключения не должны дублироваться в итоговом header.

# 0.2 Анализ AGENTS.md Codex Workflow

## Текущее состояние

Корневой `AGENTS.md` до изменения содержал только правила `graphify`. При этом `.codex/README.md`, `.codex/PROJECT_STRUCTURE.md`, `.codex/RULES.md` и `.codex/CHECKLISTS.md` уже описывали workflow проектной памяти, но агент мог не загрузить эти правила автоматически в новых задачах.

## Релевантные файлы

- `AGENTS.md`
- `.codex/README.md`
- `.codex/PROJECT_STRUCTURE.md`
- `.codex/RULES.md`
- `.codex/CHECKLISTS.md`
- `.codex/specs/0.2-agents-codex-workflow.md`
- `.codex/analysis/0.2-agents-codex-workflow-analysis.md`
- `.codex/implementation/0.2-implementation-log.md`
- `.codex/reports/0.2-final-report.md`

## Решения

- Использовать task id `0.2`, потому что это вторая meta-задача по workflow после `0.1`.
- Добавить в `AGENTS.md` короткий routing-раздел, а не переносить туда все подробные правила `.codex`.
- Оставить `graphify` раздел без изменений.
- Создать полный набор task-документов, так как пользователь прямо попросил закреплять `.codex` для новых задач и эта задача меняет постоянные инструкции агента.

## Риски

- Правила в `AGENTS.md` могут со временем разойтись с `.codex/RULES.md`, поэтому раздел должен оставаться кратким и ссылаться на authoritative `.codex` файлы.
- Для тривиальных запросов полный набор task-doc файлов может быть избыточен; формулировка ограничена нетривиальными задачами.

# Журналы Реализации

Здесь хранятся журналы реализации задач.

## Именование

- `<task-id>-implementation-log.md`, например `1.1-implementation-log.md`.
- Один `<task-id>` во всех четырёх папках: `specs/`, `analysis/`, `implementation/`, `reports/`.
- Скелет нового файла — `../templates/implementation-template.md`.

## Рекомендуемые разделы

- План
- Изменения
- Журнал Изменённых Файлов
- Проверка
- Открытые Пункты

### Каждая запись об изменённом файле должна включать

- Файл
- Причина
- Краткое описание
- Возможные побочные эффекты

## Правила

- Документируйте здесь каждое изменение кода или документации.
- Сохраняйте task-id согласованным с `specs/`, `analysis/`, `reports/`.
- Дополняйте полезную историю вместо удаления существующей документации.
- Основной текст, заголовки и пояснения — на русском; технические идентификаторы без перевода.

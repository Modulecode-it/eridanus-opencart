# 0.1 Правила Рабочего Процесса Codex

## Цель

Сохранить требования пользователя к рабочему процессу в проектной документации `.codex` для будущих задач.

## Область

- Обновить существующие правила и чеклисты `.codex`.
- По возможности сохранять документацию append-only.
- Добавить документацию уровня задачи для этой meta-задачи.
- Не проверять и не изменять запрещенные runtime/generated директории.

## Требования

- Перед работой прочитать `.codex/PROJECT_STRUCTURE.md`.
- Избегать широкого сканирования репозитория.
- Использовать только целевые директории, описанные в `PROJECT_STRUCTURE.md`.
- Не изменять `storage79/`, `system/storage/modification/`, `webstat/` или `.git/`.
- Сохранять результаты анализа в `.codex/analysis/`.
- Документировать решения и архитектурные решения.
- Документировать изменения кода и документации в `.codex/implementation/`.
- Создавать итоговый отчет в `.codex/reports/`.
- Фиксировать каждый измененный файл с файлом, причиной, кратким описанием и возможными побочными эффектами.
- Не удалять существующую документацию; дополнять или обновлять ее.
- Писать документы `.codex` на русском языке, оставляя технические идентификаторы без перевода.

## Ограничения

- Это задача только по документации.
- Код приложения OpenCart не должен изменяться.
- Секреты, логи, данные клиентов или учетные данные не должны копироваться в task docs.

## Критерии Приемки

- Постоянные правила рабочего процесса добавлены в документацию `.codex`.
- У этой задачи есть analysis, implementation log и final report файлы.
- Changed-file journal присутствует в implementation log.
- Правило русского языка закреплено для новых документов `.codex`.

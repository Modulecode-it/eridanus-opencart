# Промт: Анализ Задачи

Этап: **spec → analysis**. Исследование текущего состояния без изменения кода.

## Вход

- `.agent/CORE.md` — структура памяти и конвенции.
- `.agent/PROJECT_STRUCTURE.md` — карта репозитория.
- `.agent/STACK.md` — правила и команды стека.
- `.agent/specs/<task-id>-<slug>.md` — спецификация задачи.
- (Опционально) `graphify query "<тема>"` если подключён graphify.

## Что Сделать

1. Прочитать спецификацию и понять требования/критерии приёмки.
2. Использовать `PROJECT_STRUCTURE.md` для точечного поиска, **не сканировать весь репозиторий**.
3. Изучить текущее поведение, затронутые файлы, поток данных, риски, варианты решения.
4. Не изменять код приложения. Разрешено: читать, искать, запускать read-only команды (lint --dry-run, route:list, и т.п.).
5. Если решение выходит за рамки задачи — предложить отдельный ADR.

## Куда Сохранить

`.agent/analysis/<task-id>-<slug>-analysis.md` (скелет — `.agent/templates/analysis-template.md`).

Обязательные разделы: Текущее Поведение · Релевантные Файлы · Поток Данных · Риски · Решения · Архитектурные Решения · Результаты Анализа.

## После

- Обновить `.agent/INDEX.md` если задача новая.
- Если найдены новые долговечные знания — обновить `PROJECT_STRUCTURE.md` / `STACK.md` / `RULES.md` / `CHECKLISTS.md` согласно таблице в `CORE.md`.
- Кратко доложить пользователю: ключевые находки, риски, предложенный план реализации.

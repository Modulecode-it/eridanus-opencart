# Промт: Аудит Реализации (Итоговый Отчёт)

Этап: **implementation → report**. Аудит сделанного и фиксация результата.

## Вход

- `.agent/CORE.md` — структура отчёта.
- `.agent/specs/<task-id>-<slug>.md` — требования и критерии приёмки.
- `.agent/analysis/<task-id>-<slug>-analysis.md` — что планировалось.
- `.agent/implementation/<task-id>-implementation-log.md` — что сделано.
- `git diff` / `git status` — фактические изменения.

## Что Сделать

1. Сверить фактически изменённые файлы с журналом реализации. Если расхождения — отметить.
2. Сверить результат с критериями приёмки из спецификации. Отметить выполненные и невыполненные.
3. Проверить, что не закоммичены секреты/логи/данные клиентов.
4. Чётко разделить «выполненная проверка» и «невыполненная проверка».
5. Перечислить известные пробелы и технический долг.
6. Предложить только конкретные, естественно вытекающие следующие шаги.

## Куда Сохранить

`.agent/reports/<task-id>-final-report.md` (скелет — `.agent/templates/report-template.md`).

Обязательные разделы: Сводка · Изменённые Файлы · Проверка (выполненная/невыполненная) · Известные Пробелы · Следующие Шаги.

## После

- Обновить `.agent/INDEX.md`: статус задачи → `завершён` или `код подготовлен`, дата.
- Если обнаружены новые долговечные знания — обновить `PROJECT_STRUCTURE.md` / `STACK.md` / `RULES.md` / `CHECKLISTS.md`.
- Кратко доложить пользователю итог по чек-листу из `CHECKLISTS.md` («Финальный Ответ Пользователю»).

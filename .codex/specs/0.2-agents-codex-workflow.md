# 0.2 AGENTS.md Codex Workflow

## Цель

Закрепить в корневом `AGENTS.md` обязательное использование проектной памяти `.codex` для будущих нетривиальных задач.

## Область

- Обновить только правила для агентов и документацию этой meta-задачи.
- Не менять checkout-flow, OpenCart controller/view/model файлы или runtime-данные.
- Не дублировать полностью содержимое `.codex/RULES.md`; `AGENTS.md` должен направлять агента к существующим authoritative files.

## Требования

- Добавить раздел `## codex project memory` в `AGENTS.md`.
- Указать, что перед широким анализом или реализацией новой нетривиальной задачи агент читает `.codex/README.md`, `.codex/PROJECT_STRUCTURE.md` и `.codex/RULES.md`.
- Указать использование `.codex/CHECKLISTS.md` как рабочего checklist.
- Указать, что task-документация создается или обновляется с одним task id в `.codex/specs/`, `.codex/analysis/`, `.codex/implementation/` и `.codex/reports/`.
- Зафиксировать русский язык для новых и обновляемых документов `.codex` при сохранении технических идентификаторов без перевода.
- Зафиксировать запрет на сохранение секретов, токенов, полных логов, клиентских данных и credentials окружения в `.codex`.

## Критерии приемки

- `AGENTS.md` содержит раздел `codex project memory`.
- Существующий раздел `graphify` сохранен.
- У этой задачи есть согласованные task-документы с task id `0.2`.
- Проверка подтверждает наличие новых правил и ссылок на существующие `.codex` файлы.

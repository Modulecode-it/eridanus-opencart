# 1.3 Установка и базовая настройка Simple 4.11.9

## Цель

Подготовить точную инструкцию по установке и первичной настройке модуля Simple `4.11.9` для проекта OpenCart `3.0.3.2`. В этой задаче сам модуль не устанавливается, изменяются только документы в `.codex`.

## Что известно об архиве

Получен архив `simple 4.11.9.zip` со следующей структурой:

```text
install/
misc/
update/
changelog.txt
faq_ru.html
install_ru.html
license_ru.html
update_ru.html
```

Из этой структуры следует:

- `install/` использовать для первичной установки;
- `update/` использовать только для обновления уже установленного Simple;
- `misc/` использовать только для дополнительных шаблонов и вспомогательных файлов;
- `install_ru.html` считать главным источником пошаговой установки;
- `faq_ru.html` использовать для проверки типовых проблем и совместимости;
- `license_ru.html` прочитать до любых действий с пакетом;
- `changelog.txt` использовать как дополнительную проверку изменений версии `4.11.9`.

## Область задачи

В рамках будущей установки нужно:

- установить Simple на OpenCart `3.0.3.2`;
- использовать именно пакет из `install/upload for 3.0.x`;
- проверить наличие `ionCube Loader`;
- выполнить базовую настройку checkout;
- проверить совместимость с `filterit` и `measoftcourier`;
- обновить modifications и cache после установки.

## Что входит в инструкцию

В будущей установке должны быть описаны такие шаги:

1. Подготовить резервную копию файлов и базы данных.
2. Проверить версию OpenCart и PHP.
3. Проверить `ionCube Loader`.
4. Прочитать `license_ru.html` и `install_ru.html`.
5. Выбрать каталог `install/upload for 3.0.x`.
6. Загрузить файлы Simple в корень сайта с сохранением структуры.
7. Установить модуль через админку OpenCart, если пакет поставляется как `ocmod` или installer-архив.
8. Обновить `Extensions -> Modifications`.
9. Очистить кэш OpenCart.
10. Включить Simple в админке.
11. Выполнить базовую настройку checkout.
12. Проверить фронтенд checkout, регистрацию и оформление заказа.
13. Проверить интеграции `filterit` и `measoftcourier`.

## Ограничения

1. В этой задаче нельзя устанавливать модуль Simple в сам проект.
2. Нельзя менять файлы OpenCart, кроме будущей отдельной задачи по установке.
3. Нельзя распаковывать архив поверх репозитория без подтвержденной инструкции.
4. Нельзя сохранять лицензии, токены, ключи, пароли, полные логи и клиентские данные в `.codex`.
5. Нельзя использовать `update/` для чистой установки.
6. Нельзя считать `misc/` обязательной частью установки без проверки темы.

## Риски

- отсутствие или неподходящая версия `ionCube Loader`;
- конфликт Simple с уже установленными модулями `filterit` и `measoftcourier`;
- несовместимость с темой, если она не стандартная;
- ошибки при выборе неправильной папки `install/` или `update/`;
- необходимость очищать modifications/cache после установки;
- возможные проблемы, если checkout route отличается от `checkout/simplecheckout`.

## Критерии приемки

Документация считается готовой, если:

- в `.codex` описан один общий task id `1.3-simple-install`;
- есть понятная инструкция по установке именно из `install/upload for 3.0.x`;
- отдельно отмечены проверки `ionCube Loader`, modifications, cache, `filterit`, `measoftcourier` и checkout;
- указаны ограничения и риски для проекта;
- документы написаны на русском языке, а технические идентификаторы сохранены без перевода.

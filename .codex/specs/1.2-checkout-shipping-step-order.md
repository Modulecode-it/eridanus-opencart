# 1.2 Порядок Шагов Доставки Checkout

## Цель

Поменять местами шаги доставки на странице `checkout/checkout`: `Шаг 3` должен быть способом доставки, а `Шаг 4` адресом доставки.

## Область

- Storefront checkout OpenCart в `catalog/controller/checkout/*`.
- Основной checkout accordion в `catalog/view/theme/default/template/checkout/checkout.twig`.
- Не менять тексты языковых файлов, payment/shipping extensions и generated modification files.

## Требования

- Заголовок `text_checkout_shipping_method` должен получать номер `3`.
- Заголовок `text_checkout_shipping_address` должен получать номер `4`.
- Панель `collapse-shipping-method` должна располагаться выше `collapse-shipping-address`.
- После сохранения платежного адреса авторизованного клиента сначала открывать способ доставки, используя текущий/default адрес доставки.
- Если отдельный адрес доставки ещё не задан, использовать сохраненный платежный адрес как default `shipping_address` для расчета способов доставки.
- После изменения адреса доставки сохранять текущее поведение пересчета способов доставки и возврата пользователя к выбору способа доставки.

## Критерии приемки

- На `/checkout` при товарах с доставкой визуальный порядок: `Шаг 3: Способ доставки`, затем `Шаг 4: Адрес доставки`.
- Авторизованный checkout после сохранения платежного адреса открывает выбор способа доставки, если способы доступны.
- Guest/register сценарии с совпадающим адресом продолжают показывать способ доставки перед отдельным адресным блоком.
- Сценарии без доставки сохраняют прежние номера платежа и подтверждения.

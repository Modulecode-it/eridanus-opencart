# 0.1 Codex Workflow Rules

## Goal

Persist the user's workflow requirements for future tasks in the `.codex` project documentation.

## Scope

- Update existing `.codex` rules and checklists.
- Keep documentation append-only where possible.
- Add task-level documentation for this meta-task.
- Do not inspect or modify forbidden runtime/generated directories.

## Requirements

- Read `.codex/PROJECT_STRUCTURE.md` before work.
- Avoid broad repository scans.
- Use only target directories described by `PROJECT_STRUCTURE.md`.
- Do not change `storage79/`, `system/storage/modification/`, `webstat/`, or `.git/`.
- Save analysis results in `.codex/analysis/`.
- Document decisions and architectural decisions.
- Document code and documentation changes in `.codex/implementation/`.
- Create a final report in `.codex/reports/`.
- Record every changed file with file, reason, short description, and possible side effects.
- Do not delete existing documentation; append or update it.

## Constraints

- This is a documentation-only task.
- No OpenCart application code should change.
- No secrets, logs, customer data, or credentials should be copied into task docs.

## Acceptance Criteria

- Persistent workflow rules are added to `.codex` documentation.
- This task has analysis, implementation log, and final report files.
- Changed-file journal is present in the implementation log.

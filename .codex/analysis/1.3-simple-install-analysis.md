# 1.3 Анализ установки Simple 4.11.9

## Контекст проекта

Проект работает на OpenCart `3.0.3.2`. В проекте уже есть следы интеграций, которые завязаны на поведение Simple, поэтому установку нельзя рассматривать изолированно.

## Что показал архив

Пользователь предоставил архив `simple 4.11.9.zip` со структурой:

```text
install/
misc/
update/
changelog.txt
faq_ru.html
install_ru.html
license_ru.html
update_ru.html
```

Практический вывод для этого проекта:

- `install/` — базовый источник установки;
- `install/upload for 3.0.x` — ветка, которая подходит для OpenCart `3.0.x`;
- `update/` — только для уже установленного Simple;
- `misc/` — дополнительные шаблоны, которые могут понадобиться для нестандартной темы;
- `install_ru.html` — основная инструкция;
- `faq_ru.html` — источник типовых ошибок, требований и предупреждений;
- `license_ru.html` — обязательное чтение до любых действий.

## Что важно для проекта

В кодовой базе уже есть зависимости от элементов Simple:

- `system/library/simple/filterit.php` — кодированная библиотека интеграции `filterit` с Simple;
- `catalog/view/theme/default/template/extension/module/filterit.twig` — использует классы `simplecheckout-block` и `simplecheckout-block-content`;
- `catalog/view/javascript/measoftcourier/checkout_script.js` — ищет кнопку `#simplecheckout_button_confirm`;
- `admin/language/ru-ru/extension/module/filterit.php` — содержит настройки для поля Simple и предупреждение `invert_order_in_simple`.

Это означает, что установка Simple влияет не только на checkout, но и на уже существующие модули доставки и фильтров.

## Вывод по архиву

Для чистой установки нужно делать так:

1. Брать только `install/upload for 3.0.x`.
2. Не использовать `update/`.
3. Не переносить `misc/` автоматически, а сначала проверить, нужен ли он для темы.
4. После загрузки файлов обязательно обновить `Extensions -> Modifications` и cache.
5. Проверить работу `filterit` и `measoftcourier` после активации Simple.

## Риски

1. `ionCube Loader` может отсутствовать или не подходить по версии, и тогда кодированные части не запустятся.
2. Нестандартная тема может потребовать ручного переноса файлов из `misc/templates/<theme>/`.
3. Конфликт с текущими OCMOD-модификациями может сломать оформление заказа.
4. `measoftcourier` может перестать видеть `#simplecheckout_button_confirm`, если разметка Simple отличается от ожидаемой.
5. `filterit` может требовать конкретного порядка блоков и настроек, связанных с `invert_order_in_simple`.
6. Если установить Simple без staging-проверки, есть риск сломать checkout на рабочем сайте.

## Решение для будущей установки

- Использовать `install_ru.html` как главный runbook.
- Перед установкой проверить `ionCube Loader` и версию PHP.
- Устанавливать только после резервной копии.
- Не менять generated files вручную.
- После установки проверить страницу `checkout/simplecheckout`, гостевой checkout, авторизованный checkout, `filterit` и `measoftcourier`.

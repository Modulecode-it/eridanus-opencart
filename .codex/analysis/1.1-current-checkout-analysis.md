# 1.1 Анализ Текущего Checkout

Дата: 2026-06-10

Область: storefront checkout OpenCart 3.0.3.2, cart AJAX, checkout-related payment/shipping/total extensions, checkout events, API-adjacent checkout controllers. Generated modification directories не аудировались как primary source.

## Сводка

Checkout является стандартным OpenCart 3 accordion checkout, реализованным через `catalog/controller/checkout/*` и inline JavaScript в `catalog/view/theme/default/template/checkout/checkout.twig`.

Browser flow управляется AJAX:

1. `checkout/checkout` рендерит пустые accordion panels.
2. Страница загружает `checkout/login` для гостей или `checkout/payment_address` для авторизованных клиентов.
3. Шаги identity/address заполняют session state.
4. Shipping methods рассчитываются моделями `extension/shipping/*`.
5. Payment methods рассчитываются моделями `extension/payment/*`.
6. `checkout/confirm` строит `order_data`, создает/обновляет `session->data['order_id']` и загружает выбранный payment controller.
7. Выбранный payment controller подтверждает заказ или перенаправляет на gateway, затем страницы success/failure завершают user flow.

Важная кастомная checkout logic находится в CDEK, Measoft, `cod_cdek`, `filterit`, `sms_alert`, Yandex buy status events и voucher email events.

## Все Checkout AJAX Routes

### Основной Accordion Checkout

- `GET index.php?route=checkout/login` - загрузить guest/login account choice panel.
- `GET index.php?route=checkout/payment_address` - загрузить payment address panel для авторизованных клиентов.
- `GET index.php?route=checkout/{account}` - динамическая форма `register` или `guest` из выбранного account type.
- `POST index.php?route=checkout/login/save` - аутентифицировать клиента во время checkout.
- `POST index.php?route=checkout/register/save` - создать customer и checkout address/session data.
- `POST index.php?route=checkout/payment_address/save` - сохранить выбранный или новый payment address.
- `POST index.php?route=checkout/shipping_address/save` - сохранить выбранный или новый shipping address.
- `POST index.php?route=checkout/guest/save` - сохранить guest payment/customer data.
- `POST index.php?route=checkout/guest_shipping/save` - сохранить guest shipping data.
- `POST index.php?route=checkout/shipping_method/save` - проверить и сохранить выбранный shipping method.
- `POST index.php?route=checkout/payment_method/save` - проверить и сохранить выбранный payment method/comment/agree flag.
- `GET index.php?route=checkout/shipping_address` - загрузить shipping address panel.
- `GET index.php?route=checkout/guest_shipping` - загрузить guest shipping address panel.
- `GET index.php?route=checkout/shipping_method` - рассчитать и загрузить shipping methods panel.
- `GET index.php?route=checkout/payment_method` - рассчитать и загрузить payment methods panel.
- `GET index.php?route=checkout/confirm` - построить order data и загрузить confirm panel плюс выбранный payment UI.

### Address Helpers

- `GET index.php?route=checkout/checkout/country&country_id={id}` - country metadata и zones для address forms.
- `GET index.php?route=checkout/checkout/customfield&customer_group_id={id}` - custom fields по customer group.
- `POST index.php?route=tool/upload` - загрузка file custom field из checkout address forms.

### Cart And Header Cart

- `POST index.php?route=checkout/cart/add` - добавить product в cart из product/list pages.
- `POST index.php?route=checkout/cart/edit` - обновить cart quantity.
- `POST index.php?route=checkout/cart/remove` - удалить item из cart/header cart.

### Cart Total Extensions

- `POST index.php?route=extension/total/voucher/voucher` - применить voucher.
- `POST index.php?route=extension/total/reward/reward` - применить reward points.
- `POST index.php?route=extension/total/shipping/quote` - рассчитать shipping из cart estimator.

- `GET index.php?route=extension/total/shipping/country&country_id={id}` - country metadata для cart shipping estimator.

## Текущая Последовательность Checkout Step

### Entry Conditions

1. `checkout/checkout::index` перенаправляет в `checkout/cart`, если в cart нет products/vouchers или stock rules не проходят.
2. Проверяет minimum quantity для каждого cart product.
3. Загружает language, breadcrumbs, scripts, account mode, `shipping_required` и layout controllers.
4. Рендерит `checkout/checkout.twig`.

### Guest Not Logged In

1. Page ready загружает `checkout/login` в `collapse-checkout-option`.
2. Пользователь выбирает `register` или `guest`; `button-account` загружает `checkout/register` или `checkout/guest` в `collapse-payment-address`.
3. Если пользователь логинится, `checkout/login/save` аутентифицирует его и перенаправляет обратно в `checkout/checkout`.
4. Register path отправляет `checkout/register/save`.
5. Guest path отправляет `checkout/guest/save`.
6. Если shipping required и payment address также является shipping address, checkout загружает `checkout/shipping_method` и дополнительно обновляет `checkout/shipping_address` или `checkout/guest_shipping`.
7. Если shipping required и нужен отдельный shipping address, checkout загружает `checkout/shipping_address` или `checkout/guest_shipping`.
8. Если shipping не required, checkout загружает `checkout/payment_method`.

### Logged Customer

1. Page ready загружает `checkout/payment_address`.
2. `button-payment-address` отправляет `checkout/payment_address/save`.
3. Если shipping required, checkout загружает `checkout/shipping_address`.
4. `button-shipping-address` отправляет `checkout/shipping_address/save`, затем загружает `checkout/shipping_method`.
5. Если shipping не required, checkout загружает `checkout/payment_method`.

### Shipping Method

1. `checkout/shipping_method::index` загружает enabled shipping extensions из `model_setting_extension->getExtensions('shipping')`.
2. Для каждого active shipping extension вызывает `model_extension_shipping_{code}->getQuote($session->data['shipping_address'])`.
3. Сохраняет отсортированные results в `session->data['shipping_methods']`.
4. `button-shipping-method` отправляет selected method/comment в `checkout/shipping_method/save`.
5. При success загружает `checkout/payment_method`.
6. CDEK может перехватить `checkout/shipping_method/save/before` и потребовать выбранный PVZ.

### Payment Method

1. `checkout/payment_method::index` пересчитывает totals через enabled total extensions.
2. Загружает enabled payment extensions из `model_setting_extension->getExtensions('payment')`.
3. Для каждого active payment extension вызывает `model_extension_payment_{code}->getMethod($session->data['payment_address'], $total)`.
4. Сохраняет отсортированные results в `session->data['payment_methods']`.
5. `button-payment-method` отправляет selected method/comment/terms agreement в `checkout/payment_method/save`.
6. При success загружает `checkout/confirm`.

### Confirm And Order Creation

1. `checkout/confirm::index` проверяет shipping и payment session prerequisites.
2. Пересчитывает totals через enabled total extensions.
3. Строит `order_data` из customer/guest data, payment address, shipping address, cart products, vouchers, comments, tracking, currency, IP и user agent.
4. Вызывает `model_checkout_order->addOrder($order_data)` и пишет `session->data['order_id']`.
5. Строит display products/totals для `checkout/confirm.twig`.
6. Загружает selected payment controller через `extension/payment/{payment_method.code}`.
7. Payment template/controller сразу подтверждает order history или redirect/submit во внешний gateway.

### Success/Failure

1. Успешная оплата ведет в `checkout/success`.
2. `checkout/success::index` очищает cart, order/session checkout data, coupon/voucher/reward state.
3. CDEK success event также очищает `session->data['cdek']`.
4. Failed external payment routes могут использовать `checkout/failure`.

## Extension Points Для Новой Логики

### Low-Risk Extension Points

- Добавить OpenCart event before/after `catalog/controller/checkout/payment_method/save` для валидации payment selection.
- Добавить OpenCart event before/after `catalog/controller/checkout/shipping_method/save` для валидации shipping selection.
- Добавить OpenCart event before/after `catalog/model/checkout/order/addOrder` для enrichment order data после creation.
- Добавить OpenCart event before/after `catalog/model/checkout/order/addOrderHistory` для запуска integrations после изменений payment/order status.
- Добавить новый `catalog/model/extension/total/{code}.php` для расчета fees/discounts через existing totals pipeline.
- Добавить новый `catalog/model/extension/payment/{code}.php` и `catalog/controller/extension/payment/{code}.php` для payment-specific logic.
- Добавить новый `catalog/model/extension/shipping/{code}.php` для shipping rates с использованием `getQuote($address)`.

### Frontend Extension Points

- `catalog/view/theme/default/template/checkout/checkout.twig` является центральной client state machine. Любой новый AJAX step должен учитывать все branches: logged customer, guest, register, shipping required, shipping not required.
- `checkout/payment_method.twig` - самое безопасное место для UI, связанного с payment selection.
- `checkout/shipping_method.twig` - самое безопасное место для UI, связанного с shipping selection.
- `checkout/confirm.twig` - самое безопасное место для финального read-only order summary UI.
- Address templates могут добавлять новые fields, но соответствующие `save` controllers должны валидировать и сохранять их в session/order data.

### Session State Extension Points

- `session->data['payment_address']`
- `session->data['shipping_address']`
- `session->data['payment_methods']`
- `session->data['payment_method']`
- `session->data['shipping_methods']`
- `session->data['shipping_method']`
- `session->data['guest']`
- `session->data['comment']`
- `session->data['order_id']`
- `session->data['cdek']`

### Existing Custom Logic To Preserve

- CDEK хранит PVZ data в `session->data['cdek']`, валидирует их перед shipping method save и пишет их в `order_to_sdek` плюс order history comment.
- Доступность оплаты `cod_cdek` зависит от CDEK/shipping settings и может принудительно установить `session->data['payment_method']['code'] = 'cod_cdek'`.
- `total/cdek` добавляет или вычитает percent-based fee, когда `payment_method.code == cod_cdek`.
- Measoft JS и controller могут напрямую изменять `session->data['shipping_method']` для `measoftcouriershipping.standard`.
- `filterit` имеет payment/total logic, которая может менять costs в зависимости от selected payment method.
- `sms_alert`, Yandex buy modules и voucher total регистрируют events на `addOrderHistory/after`.

### Risk Notes

- `catalog/controller/event/cdekshipping.php` использует assignment в route checks: `if($route = "checkout/order/addOrder" && ...)`. Это нужно учитывать осторожно при изменении event logic, потому что route string не сравнивается.
- `catalog/model/extension/payment/modulbank.php`, похоже, запрашивает `payment_liqpay_geo_zone_id` при проверке доступности Modulbank. Это может быть copy-paste bug, влияющий на payment visibility.
- `catalog/model/extension/total/cdek.php` пишет полный session data в log `total_cdek` во время total calculation. Это может раскрывать operational/customer data в logs.
- `system/storage/modification/` и `storage79/modification/` могут содержать generated runtime variants. Они намеренно не использовались как source of truth.

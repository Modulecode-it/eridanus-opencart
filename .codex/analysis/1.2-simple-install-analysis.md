# 1.2 Анализ установки Simple

## Контекст проекта

Проект работает на OpenCart 3.0.3.2. Основной checkout находится в `catalog/controller/checkout/` и шаблонах `catalog/view/theme/default/template/checkout/`.

Simple в этой задаче рассматривается как сторонний коммерческий модуль упрощенного оформления заказа. Обычно он заменяет стандартный многошаговый checkout OpenCart на настраиваемую страницу с AJAX-обновлением блоков покупателя, адреса, доставки, оплаты и подтверждения заказа.

## Текущее состояние

Полная установка Simple в рабочем дереве не обнаружена по типичным файлам:

- `admin/controller/extension/module/simple.php` отсутствует;
- `catalog/controller/extension/module/simple.php` отсутствует;
- отдельные frontend assets Simple вроде `catalog/view/javascript/simplecheckout.js` не обнаружены.

При этом в проекте есть следы совместимости с Simple:

- `system/library/simple/filterit.php` - закодированная библиотека интеграции `filterit` с Simple, требует учета `ionCube Loader`;
- `catalog/view/theme/default/template/extension/module/filterit.twig` использует классы `simplecheckout-block` и `simplecheckout-block-content`;
- `catalog/view/javascript/measoftcourier/checkout_script.js` ищет кнопку `#simplecheckout_button_confirm`;
- `admin/language/ru-ru/extension/module/filterit.php` содержит настройки для кастомного поля Simple и предупреждение `invert_order_in_simple`.

Эти следы означают, что установка Simple должна проверяться не изолированно, а вместе с текущими модулями доставки, оплаты и условий.

## Риски

1. `ionCube Loader` может отсутствовать или не соответствовать версии PHP, из-за чего закодированные файлы Simple или интеграции `filterit` не выполнятся.
2. OCMOD может изменить стандартные checkout/account файлы, поэтому после установки нужно обновлять modifications и проверять конфликтующие операции.
3. `filterit` может требовать определенный порядок загрузки блоков доставки и оплаты; нужно учитывать настройку/подсказку `invert_order_in_simple`.
4. `measoftcourier` зависит от DOM-элементов Simple, включая `simplecheckout_button_confirm`; изменение разметки может сломать блокировку заказа без выбранного ПВЗ.
5. Установка на production без staging-проверки может нарушить оформление заказа.
6. Лицензионные данные Simple нельзя фиксировать в `.codex`, Git, логах или примерах конфигурации.

## Решения для будущей установки

- Использовать только официальный пакет Simple для OpenCart 3.x, предоставленный владельцем проекта.
- Сначала проверить `ionCube Loader`, затем устанавливать модуль.
- Устанавливать через OpenCart `Extensions -> Installer`, если пакет поставлен как `.ocmod.zip`.
- После установки обновлять `Extensions -> Modifications` и очищать cache.
- Базовую настройку выполнять минимально: включить checkout, необходимые поля, блоки доставки/оплаты/подтверждения.
- Проверять полный заказ с `filterit` и `measoftcourier`, а не только загрузку страницы checkout.

## Дополнение: реальная установка 4.11.9 и починка замены ссылок (2026-07-14)

Файлы модуля Simple 4.11.9 залиты из `install/upload for 3.0.x/` (admin, catalog,
system) в корень проекта. Возникла ошибка кнопки «Протестировать замену» в админке
Simple: «Не работает замена ссылок, посмотрите справку».

### Как устроена замена ссылок в OpenCart 3.x

Simple не правит маршруты напрямую. Замена стандартных ссылок OpenCart
(`checkout/checkout`, `account/register`, `checkout/cart`) на страницы модуля
(`checkout/simplecheckout`, `account/simpleregister`, `checkout/simplecheckout`)
выполняется классом-rewrite для объекта `Url`:

- Класс: `system/library/simple/rewrite.php`, `namespace Simple; class Rewrite`.
- Метод `Simple\Rewrite::rewrite($url)` подменяет маршруты по флагам
  `simple_replace_cart`, `simple_replace_checkout`, `simple_replace_register`.
- Внутри есть защита: при отсутствии сохранённых настроек
  (`!$this->config->get('simple_settings')`) метод возвращает ссылку без изменений.

Регистрируется rewrite одной строкой после создания объекта `Url`:

```php
$this->url->addRewrite(new Simple\Rewrite($this->config, $this->session));
```

### Найденная причина

- Класс `Simple\Rewrite` присутствует и читаем (не ionCube).
- Исходный `catalog/controller/startup/startup.php` регистрировал `Url`, но
  `addRewrite()` отсутствовал.
- Modification-копии `system/storage/modification/catalog/controller/startup/startup.php`
  нет → модификатор `simple url rewrite` не применялся (его нет в БД).
- `.ocmod.xml` файлы в `system/` (`simple_twig_fix.ocmod.xml` и др.) в БД не
  установлены — twig_fix тоже не применён в modification-копии `twig.php`.

### Архитектурный конфликт modification ↔ git/deploy

Каталог `system/storage/modification/` отслеживается git и НЕ входит в
`.gitignore` — modification-копии деплоятся на dev-сервер через git, а не
генерируются OpenCart на сервере. Поэтому нажатие «Обновить» в админке даёт лишь
временный эффект: следующий deploy перетрёт результат. Принято решение
(владельцем проекта) оставить modification в git.

### Проверка автозагрузчика

`system/startup.php:74`, функция `library($class)`:
`strtolower($class)` с заменой `\` на `/` → путь
`system/library/simple/rewrite.php`. Файл существует, namespace и имя класса
совпадают. В момент строки регистрации `Url` доступны `$this->config` и
`$this->session`.

### Принятое решение

Ручная правка исходного `catalog/controller/startup/startup.php`: добавить
сразу после `registry->set('url', ...)` строку
`$this->url->addRewrite(new Simple\Rewrite($this->config, $this->session));`.
Это официальный резервный путь из `install_ru.html`, не зависит от
modification-системы и БД, попадает на dev-сервер через deploy. Безопасность —
за счёт внутренней защиты класса (без сохранённых настроек rewrite ничего не
меняет).

### Права на сервере Ubuntu (доп. требование)

Веб-сервер (`www-data`) должен владеть каталогом `system/storage/`. Не было
первопричиной ошибки rewrite, но требуется для modification-системы и кэшей:

```bash
sudo chown -R www-data:www-data /var/www/site/system/storage
sudo chown -R www-data:www-data /var/www/site/image
sudo find /var/www/site/system/storage -type d -exec chmod 755 {} \;
sudo find /var/www/site/system/storage -type f -exec chmod 644 {} \;
```

### Риски

- Правка исходного ядра может потеряться при обновлении OpenCart — отмечено в
  `RULES.md` как исключение.
- Модификатор `simple url rewrite` всё ещё может отсутствовать в БД; rewrite
  держится только на правке исходника.
- Не тестировалось вживую на dev-сервере (локально нет PHP/БД).

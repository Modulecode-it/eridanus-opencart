# 1.3 Инструкция и журнал будущей установки Simple 4.11.9

## Назначение

Этот документ фиксирует пошаговый порядок будущей установки Simple `4.11.9` из архива `simple 4.11.9.zip`. Сейчас выполняется только документация, не установка.

## Подготовка архива

1. Получить архив `simple 4.11.9.zip` от владельца проекта.
2. Проверить, что в архиве есть каталоги `install/`, `misc/`, `update/` и файлы `install_ru.html`, `faq_ru.html`, `license_ru.html`, `changelog.txt`, `update_ru.html`.
3. Открыть `license_ru.html` и убедиться, что условия использования понятны до копирования файлов.
4. Открыть `install_ru.html` и использовать его как основной источник порядка действий.
5. Если нужно подтвердить ограничения и типовые ошибки, свериться с `faq_ru.html`.

## Проверка совместимости

1. Проверить версию OpenCart. Для этой задачи ожидается `3.0.3.2`.
2. Проверить версию PHP на сервере.
3. Проверить наличие `ionCube Loader` и его совместимость с PHP.
4. Подтвердить, что сайт разворачивается на staging или local-копии, а не сразу на production.
5. Сделать резервную копию файлов и базы данных.

## Выбор ветки установки

1. Для OpenCart `3.0.x` брать каталог `install/upload for 3.0.x`.
2. Не использовать `update/`, если это первая установка Simple.
3. Если в `install/` есть `.ocmod.zip`, установить его через `Extensions -> Installer`.
4. Если в `install/` лежит структура `admin/`, `catalog/`, `system/`, загружать её в корень сайта только по инструкции из `install_ru.html`.

## Установка файлов

1. Распаковать нужную ветку `install/upload for 3.0.x` во временный каталог вне репозитория.
2. Сравнить структуру файлов с корнем OpenCart.
3. Загрузить файлы в соответствующие каталоги `admin/`, `catalog/`, `system/`.
4. Не редактировать `system/storage/modification/` вручную.
5. Не переносить `update/` для чистой установки.

## Действия в админке OpenCart

1. Зайти в `Extensions -> Installer`, если в пакете есть installer-архив.
2. Зайти в `Extensions -> Extensions -> Modules`.
3. Найти Simple и установить его.
4. Включить модуль Simple.
5. Настроить права доступа для группы администратора, если в меню появились новые route.
6. Перейти в `Extensions -> Modifications` и выполнить `Refresh`.
7. Очистить cache шаблона и системный cache средствами админки.

## Базовая настройка Simple

1. Включить checkout Simple.
2. Проверить, что активны блоки покупателя, адреса, доставки, оплаты, комментария и подтверждения заказа.
3. Настроить обязательные поля: имя, телефон, email.
4. Оставить адресные поля, которые нужны для доставки: город, адрес, postcode при необходимости.
5. Проверить маршрут checkout, обычно `checkout/simplecheckout`.
6. Сохранить настройки.

## Проверка шаблонов

1. Если тема нестандартная, проверить каталоги из `misc/templates/<theme>/`.
2. При необходимости перенести `simple_header.tpl`, `simple_footer.tpl` и `simple.css` в папки темы вручную.
3. Не применять эти файлы автоматически без сверки с текущей темой.

## Проверка совместимости с проектом

1. Открыть checkout на витрине.
2. Проверить, что `filterit` отображается внутри Simple checkout.
3. Проверить, что `measoftcourier` видит кнопку `#simplecheckout_button_confirm`.
4. Проверить выбор ПВЗ и подтверждение заказа без выбранного пункта, если это требуется логикой доставки.
5. Проверить гостевой заказ.
6. Проверить заказ авторизованного пользователя.
7. Проверить записи в OpenCart error log после теста.

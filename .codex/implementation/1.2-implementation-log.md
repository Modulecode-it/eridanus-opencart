# 1.2 Журнал реализации: замена ссылок Simple (url rewrite)

Дата: 2026-07-14

## Изменённые файлы

### `catalog/controller/startup/startup.php`

- **Причина:** кнопка «Протестировать замену» в админке Simple падала, потому что
  класс `Simple\Rewrite` не регистрировался как rewrite для объекта `Url`, а
  модификатор `simple url rewrite` отсутствует в БД и modification-копия
  `startup.php` не генерируется.
- **Краткое описание:** сразу после
  `$this->registry->set('url', new Url(...))` добавлена строка
  `$this->url->addRewrite(new Simple\Rewrite($this->config, $this->session));`.
- **Побочные эффекты:** замена ссылок начнёт работать только после сохранения
  настроек в админке Simple (внутренняя защита класса). Пока настройки не
  сохранены, поведение ссылок не меняется. На стандартные страницы
  `checkout/checkout` (когда открыты напрямую по их route) rewrite не влияет.

## Не изменённые файлы (намеренно)

- `system/storage/modification/catalog/controller/startup/startup.php` — не
  создаётся, так как правка сделана в исходнике. Каталог modification деплоится
  через git; при следующем deploy правка исходника подхватится сама.
- `.ocmod.xml` в `system/` — не тронуты. Модификаторы в БД не установлены, в
  рамках задачи не регистрировались.
- `system/library/simple/rewrite.php` — класс уже содержал нужную логику и защиту.

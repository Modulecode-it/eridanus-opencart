# 1.2 Журнал Реализации

## План

1. Зафиксировать требования и анализ в `.codex`.
2. Поменять нумерацию shipping steps в `checkout.php`.
3. Переставить panels в `checkout.twig`.
4. Обновить переход после `payment_address/save`, чтобы сначала открывался способ доставки.
5. Добавить default `shipping_address` из `payment_address` для авторизованного checkout.
6. Проверить синтаксис измененных PHP-файлов и просмотреть diff.

## Изменения

| Файл | Причина | Изменение | Возможные побочные эффекты |
| --- | --- | --- | --- |
| `catalog/controller/checkout/checkout.php` | Нумерация шагов должна соответствовать новому порядку. | `shipping_method` стал шагом `3`, `shipping_address` стал шагом `4`. | Только изменение отображаемых заголовков. |
| `catalog/controller/checkout/payment_address.php` | `shipping_method` требует `shipping_address` в session. | При отсутствии отдельного адреса доставки платежный адрес используется как default `shipping_address`. | Первичный расчет доставки для авторизованного клиента идет по платежному адресу до изменения отдельного адреса доставки. |
| `catalog/view/theme/default/template/checkout/checkout.twig` | Изменить порядок accordion и переход после payment address. | Панель способа доставки поднята выше адреса; после payment address открывается способ доставки, адрес загружается в фоне. | Требуется браузерная проверка полного checkout. |
| `.codex/specs/1.2-checkout-shipping-step-order.md` | Зафиксировать требования. | Создана спецификация задачи. | Нет влияния на runtime. |

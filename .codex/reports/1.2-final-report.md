# 1.2 Итоговый отчёт: замена ссылок Simple (url rewrite)

Дата: 2026-07-14

## Результат

Ошибка кнопки «Протестировать замену» в админке Simple («Не работает замена
ссылок...») обусловлена тем, что rewrite-класс `Simple\Rewrite` не был
зарегистрирован для объекта `Url`. Модификатор `simple url rewrite` в БД
отсутствует, modification-копия `startup.php` не генерируется. Из-за того, что
`system/storage/modification/` отслеживается git и деплоится на dev-сервер,
полагаться на modification-систему нельзя — применена правка исходника.

## Изменённые файлы

- `catalog/controller/startup/startup.php` — добавлена регистрация
  `Simple\Rewrite` как rewrite для `Url`.

## Что нужно сделать на dev-сервере

1. Задеплоить изменение (правка `catalog/controller/startup/startup.php`).
2. Выдать права веб-серверу на каталоги записи (см. ниже).
3. Зайти в админку Simple, **сохранить настройки модуля** (это запишет
   `simple_settings` и флаги `simple_replace_*` в БД — без них rewrite
   остаётся неактивным по защите класса).
4. Нажать «Протестировать замену» — должно загореться зелёным.
5. Проверить с главной страницы сайта, что ссылки «Регистрация» и «Оформление
   заказа» ведут на `account/simpleregister` и `checkout/simplecheckout`.

## Права на сервере Ubuntu

Веб-сервер (`www-data` для nginx+php-fpm / apache) должен владеть каталогом
записи `system/storage/`:

```bash
sudo chown -R www-data:www-data /var/www/site/system/storage
sudo chown -R www-data:www-data /var/www/site/image
sudo find /var/www/site/system/storage -type d -exec chmod 755 {} \;
sudo find /var/www/site/system/storage -type f -exec chmod 644 {} \;
```

Каталоги, где OpenCart обязан иметь право записи:
`system/storage/{modification,cache,logs,session,upload,download}/`, `image/`.

## Проверка

- Выполнено: статический анализ кода, проверка автозагрузчика, проверка
  отсутствия modification-копии `startup.php`, проверка защиты класса
  `Simple\Rewrite`.
- Не выполнено: `php -l` (PHP недоступен локально), живое тестирование на
  dev-сервере.

## Известные пробелы

- Модификатор `simple url rewrite` по-прежнему может отсутствовать в БД; это
  нормально, rewrite теперь держится на правке исходника.
- `simple_twig_fix.ocmod.xml` в `system/` не установлен в БД и не применён —
  отдельная задача, если понадобятся twig-фиксы Simple.
- Правка исходного ядра — отметить в `RULES.md` как исключение, чтобы не
  потерять при обновлении OpenCart.

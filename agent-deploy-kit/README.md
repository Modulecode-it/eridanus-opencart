# Agent Deploy Kit

Универсальный кит для развёртывания агентской системы памяти `.agent/` в любом проекте. Стек-агностичный: правила конкретного стека (OpenCart, Laravel, React, Python, …) подключаются одним файлом `STACK.md`.

**Версия кита:** 1.0.0 (см. `CHANGELOG` в развёрнутом `.agent/`).

## Что Внутри

```text
agent-deploy-kit/
├── README.md              ← этот файл
├── DEPLOY.md              ← ★ мастер-промт развёртывания (главный файл!)
├── core/                  ← готовое универсальное ядро .agent/
│   ├── README.md  CORE.md  RULES.md  CHECKLISTS.md
│   ├── STACK.example.md   ← пустой шаблон слота под стек
│   ├── INDEX.md  CHANGELOG.md  ONBOARDING.md
│   ├── hooks.json.example
│   ├── specs/  analysis/  implementation/  reports/  decisions/
│   ├── templates/         ← пустые скелеты документов (spec/analysis/impl/report/adr/...)
│   ├── prompts/           ← инструкции-промты агенту (analyze/implement/audit)
│   └── skills/            ← пустой слот под опциональные скиллы
├── stacks/                ← готовые адаптеры стека
│   ├── opencart.md        ← OpenCart 3.0.x
│   └── laravel.md         ← Laravel 10/11
└── optional/
    └── graphify/          ← опциональный плагин knowledge graph
        ├── README.md
        ├── skill/         ← SKILL.md + references/
        └── hooks.json.example
```

## Принцип Архитектуры

**4-документный цикл на задачу**, связываемый единым task-id:

```text
spec → analysis → implementation log → final report
```

Плюс ADR в `decisions/`, шаблоны в `templates/`, промты в `prompts/`, опциональные скиллы в `skills/`, и корневой `AGENTS.md` как компактная точка входа.

**Всё, кроме `STACK.md` и `PROJECT_STRUCTURE.md`, одинаково для любого проекта.** Смена стека = замена одного файла `STACK.md`.

## Как Использовать

### Развёртывание в новом проекте

1. Скопируй эту папку `agent-deploy-kit/` в корень целевого проекта (или используй как submodule).
2. Открой `agent-deploy-kit/DEPLOY.md` — это мастер-промт.
3. Дай агенту инструкцию «выполни инструкции в `agent-deploy-kit/DEPLOY.md`».
4. Агент развернёт `.agent/`, выберет `STACK.md` под твой стек, сгенерирует `PROJECT_STRUCTURE.md`, перенесёт существующие скиллы.
5. Проверь результат, закоммить.

Подробности — в `DEPLOY.md` и `core/ONBOARDING.md`.

### Что улучшено vs предыдущей версии (`.codex`)

- **Разделение ядра и стека**: универсальные правила в `CORE.md`/`RULES.md`/`CHECKLISTS.md`, стек-специфика — в `STACK.md`. Перенос на новый стек = 1 файл.
- **Имя `.agent/`** вместо привязки к инструменту (`.codex`).
- **`INDEX.md`** — реестр всех задач со статусами.
- **`CHANGELOG.md`** — история эволюции самой системы памяти.
- **`ONBOARDING.md`** — пошаговый гайд внедрения.
- **`templates/`** — пустые скелеты документов (раньше были только промты).
- **`prompts/`** — отделены от шаблонов, переименованы в инструкции.
- **`decisions/README.md`** — конвенция ADR.
- **`skills/README.md`** — слот под скиллы с правилами аккуратного переноса.
- **graphify** вынесен в `optional/` — подключается только где нужен.

## Как Развивать Кит Между Проектами

Кит задуман как **единый источник истины** для всех проектов:

1. Улучшения универсального ядра вноси в `core/` кита.
2. Новые адаптеры стека добавляй в `stacks/<name>.md`.
3. Новые опциональные плагины — в `optional/<name>/`.
4. Версия кита — в `CHANGELOG.md` развёрнутого `.agent/` и в этом README.
5. Для переноса улучшений в существующий проект: `diff` папок `core/`, применить изменения в `.agent/`, закоммитить с пометкой в `CHANGELOG.md`.

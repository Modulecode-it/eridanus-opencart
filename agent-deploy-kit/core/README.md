# .agent — Система Памяти Агента

Папка `.agent/` — это **долговременная память** агентской системы для проекта. Универсальное ядро + подключаемый адаптер стека + карта конкретного репозитория.

## Порядок Работы

Перед широкой работой читать в порядке:

1. **`CORE.md`** — как устроена система памяти (4-документный цикл на задачу).
2. **`PROJECT_STRUCTURE.md`** — карта конкретного репозитория.
3. **`STACK.md`** — правила и команды конкретного стека (OpenCart / Laravel / …).
4. **`RULES.md`** — универсальные правила (язык, git, секреты).
5. **`CHECKLISTS.md`** — повторяемые чек-листы.
6. Документ задачи в `specs/` (если работаем по существующей задаче).

## Структура

```text
.agent/
├── README.md              ← этот файл (навигация)
├── CORE.md                ← универсальное ядро: 4-документный цикл, task-id, имена файлов
├── RULES.md               ← универсальные правила (язык, git, безопасность)
├── CHECKLISTS.md          ← универсальные чек-листы
├── STACK.md               ← ПРАВИЛА СТЕКА (единственный стек-зависимый файл ядра)
├── PROJECT_STRUCTURE.md   ← КАРТА КОНКРЕТНОГО РЕПО (уникальна)
├── INDEX.md               ← реестр всех задач со статусами
├── CHANGELOG.md           ← история эволюции самой системы памяти
├── ONBOARDING.md          ← гайд внедрения в новый проект
├── hooks.json.example     ← опциональный пример хуков (graphify и др.)
├── specs/                 ← требования задач
├── analysis/              ← исследование задач
├── implementation/        ← журналы реализации
├── reports/               ← итоговые отчёты
├── decisions/             ← архитектурные решения (ADR)
├── templates/             ← пустые скелеты документов
├── prompts/               ← инструкции-промты для агента
└── skills/                ← опциональные скиллы (graphify, кастомные)
```

## Принцип

Каждая нетривиальная задача → **4 документа** под одним task-id:

```text
spec → analysis → implementation log → final report
```

Подробности — в `CORE.md`.

## Язык

Все новые и обновляемые документы — **на русском**. Технические идентификаторы (пути, классы, команды, SQL, XML, код) — без перевода.

## Быстрая Справка

- **Новая задача**: придумай `task-id` (`0.x` служебная / `1.x+` продуктовая), создай `specs/<id>-<slug>.md`, добавь строку в `INDEX.md`.
- **Существующая задача**: найди её в `INDEX.md`, открой 4 документа по task-id.
- **Завершение задачи**: обнови `reports/<id>-final-report.md`, поставь статус в `INDEX.md`, обнови долговечные знания если нужно.

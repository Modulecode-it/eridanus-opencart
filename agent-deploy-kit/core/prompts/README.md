# Промты-Инструкции для Агента

В отличие от `../templates/` (пустые скелеты), здесь лежат **инструкции агенту** — что прочитать, что сделать, куда сохранить результат для каждого этапа 4-документного цикла.

## Промты

| Промт | Этап цикла | Что делает |
|---|---|---|
| `analyze.md` | spec → analysis | Изучает spec и проект, проводит исследование, пишет `analysis/`. |
| `implement.md` | analysis → implementation | Реализует требования, пишет `implementation/` журнал. |
| `audit.md` | implementation → report | Аудирует реализацию, пишет `reports/` итог. |

## Использование

Открой нужный промт в начале этапа и следуй инструкции. Промты самодостаточны — содержат список что читать и куда сохранять.

## Правила

- Промты универсальны (не зависят от стека). Стек-специфика подгружается из `../STACK.md`.
- Не редактируйте промты под конкретную задачу.
- Если промт устарел относительно `CORE.md` — обновите и зафиксируйте в `../CHANGELOG.md`.

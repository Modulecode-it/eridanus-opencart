# Спецификации Задач

Здесь хранятся спецификации задач (требования).

## Именование

- `<task-id>-<slug>.md`, например `1.1-checkout-flow.md`.
- Один `<task-id>` во всех четырёх папках: `specs/`, `analysis/`, `implementation/`, `reports/`.
- Скелет нового файла — `../templates/spec-template.md`.

## Обязательные разделы

- Цель
- Область
- Требования
- Ограничения
- Критерии Приёмки

## Правила

- Фиксируйте требования пользователя перед реализацией, если задача нетривиальная.
- Сохраняйте task-id согласованным с `analysis/`, `implementation/`, `reports/`.
- Дополняйте полезные обновления вместо удаления существующей документации.
- Основной текст, заголовки и пояснения — на русском; технические идентификаторы без перевода.
- Не храните секреты, токены, данные клиентов, креды.

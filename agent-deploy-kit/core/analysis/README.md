# Анализ Задач

Здесь хранятся заметки исследования задач.

## Именование

- `<task-id>-<slug>-analysis.md`, например `1.1-current-checkout-analysis.md`.
- Один `<task-id>` во всех четырёх папках: `specs/`, `analysis/`, `implementation/`, `reports/`.
- Скелет нового файла — `../templates/analysis-template.md`.

## Рекомендуемые разделы

- Текущее Поведение
- Релевантные Файлы
- Поток Данных
- Риски
- Решения
- Архитектурные Решения
- Результаты Анализа

## Правила

- Сохраняйте все результаты анализа задач в этой папке.
- Фиксируйте решения и архитектурные решения, если они влияют на реализацию.
- Не храните секреты, полные логи, данные клиентов, креды.
- Дополняйте полезные находки вместо удаления существующей документации.
- Основной текст, заголовки и пояснения — на русском; технические идентификаторы без перевода.
